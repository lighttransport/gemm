Qwen-Image FP4 W4A4 OMMA

Fans one prompt across the checked variants on the CUDA backend and renders each image + timing — so you can compare native 4-bit (NVFP4 SVDQuant, sm_120a block-scale mma.sync) against FP8/BF16 on the same card.
The ours-cuda runner (cuda/qimg/test_cuda_qimg) auto-selects its path from the checkpoint's keys: a Nunchaku NVFP4 OMMA dit runs the native W4A4 FP4 GEMM, an FP8 dit runs the FP8 path — all sharing the same VAE + text encoder, so only the DiT differs. The default ./server/run-qwen-compare.sh registers three: fp4 (native W4A4), fp8 (canonical qwen_image_fp8_e4m3fn), and fp8-repack (Nunchaku→dense FP8). Each generation is a one-shot process (weights reload per request), so expect tens of seconds per variant.
