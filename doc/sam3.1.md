## What Landed

- SAM3.1 ViT MLP now uses exact erf-GELU to match PyTorch default GELU.
- `precision=fp32` / `SAM3_PRECISION=fp32` uploads linear weights as F32
  and routes linear projections through `gemm_tiled_f32_f32`.
- DETR prompt construction includes the text-only geometry CLS token, so
  prompt length is `32 text tokens + 1 geometry token`.
- Decoder norm loading follows Meta's module order:
  `norm2` for self-attention, `catext_norm` for text cross-attention,
  `norm1` for image cross-attention, and `norm3` for MLP.
- `verify_final` compares the best matching CUDA mask against reference
  mask #0 instead of assuming CUDA query #0 is the top query.
- `verify_mask` consumes raw `segmentation_head_out` dumps when present.

## Build

```bash
make -C cuda/sam3.1 all
```

This builds `test_cuda_sam3_1` and the `verify_*` binaries.

## Reference Dumps

```bash
python3 ref/sam3.1/gen_image_ref.py \
  --ckpt /mnt/disk01/models/sam3.1/sam3.1_multiplex.pt \
  --image /home/syoyo/work/gemm/main/fujisan.jpg \
  --phrase mountain \
  --refdir /tmp/sam3.1_ref \
  --device cuda
```

Use a different `--phrase` and `--refdir` for regression prompts, for
example `/tmp/sam3.1_ref_snow` and `/tmp/sam3.1_ref_sky`.

## Remaining Work

- Speed: replace the custom ViT GEMM path with cuBLAS/cuBLASLt, then
  revisit Flash-Attention and launch fusion. See `cuda/sam3.1/PERF.md`.
- Precision variants: real bf16/fp8 kernels are still placeholders.
- Coverage: expand prompt/image regressions beyond mountain, snow, and
  sky.
- Product wiring: the standalone CUDA runner is current; server/web
  SAM3.1 routing remains a separate integration task.
- Scope: tracker/video and prompt-driven `interactive_convs.*` /
  `propagation_convs.*` paths are still out of scope for this image-only
  text-prompt runner.
