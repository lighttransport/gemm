# Vulkan RDNA4 WMMA GEMM

Experimental Vulkan cooperative-matrix GEMM benchmark for RDNA4-style shapes.

The path supports FP16 and BF16 inputs with FP32 accumulation:

```text
Y[M,N] = X[M,K] * W[N,K]^T
```

This matches the ROCm GEMM benchmark convention where activations are `X[M,K]`
and weights are pre-transposed as `W[N,K]`.

## Build

From the repository root:

```sh
cmake -S vulkan -B vulkan/build
cmake --build vulkan/build -j
```

Or standalone:

```sh
cmake -S vulkan/wmma -B vulkan/wmma/build
cmake --build vulkan/wmma/build -j
```

## Run

From `vulkan/build`:

```sh
./wmma/bench_wmma_gemm --check-caps
./wmma/bench_wmma_gemm --dtype bf16 --shape mm0 --iters 200 --check
./wmma/bench_wmma_gemm --dtype bf16 --shape all --iters 200 --check
```

From another working directory, pass the build directory:

```sh
vulkan/build/wmma/bench_wmma_gemm --shader-dir vulkan/build --dtype bf16 --shape mm0
```

## Notes

- The shader uses `VK_KHR_cooperative_matrix` with 16x16x16 subgroup tiles.
- The default BF16-tuned workgroup computes a 128x128x48 tile. Shapes are
  padded internally to `M % 128 == 0`, `N % 128 == 0`, `K % 48 == 0`.
- On RX 9070 XT / RADV GFX1201, the tuned BF16 `mm0` path measured about
  75-77 TFLOP/s with `--no-check` and about 73 TFLOP/s with sampled checking.
- Verification is sampled by default so large shapes such as `mm0` do not spend
  minutes in a full CPU reference.
- BF16 is generated by compiling the FP16 GLSL shader and patching the SPIR-V
  16-bit float type to `SPV_KHR_bfloat16`. Current `glslc` cannot spell the
  BF16 cooperative-matrix type directly.
